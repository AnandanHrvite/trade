☆ 0. What's New in v4.2.0 (2026-04-20)

Key infrastructure and UI upgrades added since v4.1.0. Setup steps below reflect the current system.

  • New routes: /consolidation (cross-mode trade history), /pnl-history (broker-wise realised P&L with one-time baselines + live overlay).
  • Live NIFTY candlestick chart on all paper + live status pages (Swing / Scalp / PA). Toggleable via CHART_ENABLED env var.
  • Dashboard Start-All buttons launch every enabled mode (Swing + Scalp + PA) with one click.
  • Telegram restructured: 17 per-mode toggles (TG_{SWING|SCALP|PA}_{STARTED|ENTRY|EXIT|SIGNALS|DAYREPORT}) + master TG_ENABLED gate + consolidated 15:30 IST day report (TG_DAYREPORT_CONSOLIDATED).
  • Settings UI: bulk paste config (KEY=VALUE pairs), one-click server restart, frozen dependent fields (VIX / Scalp section auto-disable when their master toggle is off).
  • Scalp V4 quality filters exposed in Settings UI: SCALP_REQUIRE_APPROACH, SCALP_MIN_BODY_RATIO, SCALP_TRAIL_GRACE_SECS.
  • Login cookie now uses SameSite=Lax (was Strict) to fix mobile OAuth redirect loop during broker login.
  • New persistent file: ~/trading-data/historical_pnl.json holds one-time P&L baselines per broker.

🔗 2. Fyers API Registration

Create Your App

  1. Go to https://myapi.fyers.in and log in with your Fyers credentials.
  2. Click Create App on the dashboard.
  3. Fill in the form:
    FieldValue
    App TypeWeb
    App NameAny descriptive name (e.g., PalaniBot)
    Redirect URLhttps://YOUR_EC2_IP:3000/auth/callback
  4. After creation, copy the APP_ID and SECRET_KEY into your .env file.

Token Lifecycle

  • Token is stored at ~/trading-data/.fyers_token
  • Auto-validates daily using IST date check
  • Fallback: token is considered expired after 20 hours of age
  • Re-authentication required each trading day before market open
Note: The Fyers token is used for both data feed (WebSocket, candles) and order placement for Scalp and Price Action strategies. Strategy 1 live orders go through Zerodha.

🔑 3. Zerodha Kite Connect Registration

Create Your App

  1. Go to https://developers.kite.trade and log in.
  2. Click Create App (Kite Connect type).
  3. Fill in the details:
    FieldValue
    App NameAny name (e.g., PalaniTrader)
    Redirect URLhttps://YOUR_EC2_IP:3000/auth/zerodha/callback
  4. Copy ZERODHA_API_KEY and ZERODHA_API_SECRET into your .env file.

~/.zerodha_token trading-data dir

Token Lifecycle

  • Token is stored at ~/trading-data/.zerodha_token
  • Expires at 6:00 AM IST the next day (Zerodha policy)
  • Must re-authenticate every trading day before market open
Important: Zerodha tokens expire strictly at 6 AM IST. Unlike Fyers, there is no age-based fallback — the token is invalidated server-side by Zerodha at the fixed time.

🔒 4. SSL Certificate Setup

Both Fyers and Zerodha OAuth require HTTPS redirect URLs. Generate a self-signed certificate for your EC2 instance:

# Create the certs directory in the project root
mkdir -p certs

# Generate a 4096-bit RSA self-signed certificate (valid for 10 years)
openssl req -x509 -newkey rsa:4096 \
  -keyout certs/key.pem -out certs/cert.pem \
  -days 3650 -nodes -subj "/CN=YOUR_EC2_IP"
FilePurposeLocation
certs/key.pemPrivate key (RSA 4096-bit)Project root → certs/
certs/cert.pemSelf-signed X.509 certificateProject root → certs/
Security: Never commit certs/ to version control. Ensure it is listed in .gitignore. The private key must remain on the server only.

💬 6. Telegram Bot Setup

Create the Bot

  1. Open Telegram and search for @BotFather.
  2. Send /newbot and follow the prompts to name your bot.
  3. Copy the BOT_TOKEN provided by BotFather.
  4. Open a conversation with your new bot and send any message (e.g., /start).
  5. Get your Chat ID by calling the Telegram API: 
    curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates" | jq '.result[0].message.chat.id'
  6. Set both values in your .env file.

🔌 8. Auth Status Routes

Use these endpoints to check broker authentication status and manage tokens.

MethodRouteDescription
GET /auth/status Check Fyers authentication status. Returns token validity and expiry info.
GET /auth/zerodha/status Check Zerodha Kite Connect authentication status.
GET /auth/status/all Combined status for both Fyers and Zerodha brokers in a single response.
GET /auth/zerodha/logout Clear Zerodha token and force re-authentication.

Login Endpoints

MethodRouteDescription
GET /auth/login Initiates Fyers OAuth flow. Redirects browser to Fyers login page.
GET /auth/zerodha/login Initiates Zerodha OAuth flow. Redirects browser to Kite login page.
Daily Workflow: Before market open each trading day, visit /auth/status/all to verify both tokens are valid. If expired, use the respective /login endpoints to re-authenticate.

🗃 9. Data Storage

All persistent data is stored in ~/trading-data/, which lives outside the project directory. This ensures data survives code deployments, git pulls, and PM2 restarts.

File / DirectoryPurposeLifecycle
~/trading-data/.fyers_token Fyers OAuth access token Refreshed daily via /auth/login
~/trading-data/.zerodha_token Zerodha Kite access token Expires at 6 AM IST daily
~/trading-data/trade-logs/ Strategy 1 & Scalp trade history (JSON) Persistent, one file per session
~/trading-data/backtest/ Backtest results and reports Persistent
~/trading-data/cache/ Cached historical candle data Auto-pruned after CACHE_MAX_DAYS

Directory Structure

~/trading-data/
  |
  |-- .fyers_token         # Fyers access token
  |-- .zerodha_token       # Zerodha access token
  |-- trade-logs/          # Trade history files
  |-- backtest/            # Backtest result files
  |-- cache/               # Candle data cache
Backup: The ~/trading-data/ directory contains all your trade history and tokens. Back it up regularly, especially before any EC2 instance changes or migrations.
