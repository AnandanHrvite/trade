01 Strategy Overview

SCALP_BB_RSI_V4 is a 5-minute timeframe scalping strategy for NIFTY index options (CE/PE). It identifies short-term momentum breakouts using Bollinger Bands and RSI, with Parabolic SAR providing dynamic trailing stop-loss management. V4.2 adds opt-in quality filters (approach + body strength) and a trail grace period.

Core Concept

When price breaks above the upper Bollinger Band with RSI confirming bullish momentum (>55), buy a NIFTY ATM Call option. When price breaks below the lower band with bearish RSI (<45), buy an ATM Put option. PSAR dynamically trails the stop loss — tightening only, never widening — to capture the momentum move while protecting profits.

02 Indicators Used

Bollinger Bands (BB) — Volatility Envelope

Parameters: Period 20, StdDev 1.0

Bollinger Bands create a dynamic price envelope. The upper and lower bands represent 1 standard deviation from the 20-period SMA. Price breaking outside these bands indicates a momentum breakout.

BB Squeeze Filter

When BB width is below 0.15% of price (approx <36pts on NIFTY 24000), entries are skipped. Narrow bands indicate consolidation where breakouts are unreliable. Controlled via SCALP_BB_SQUEEZE_FILTER and SCALP_BB_MIN_WIDTH_PCT.

Trend Filter v4.3.0

Even when BB / RSI both fire, the trend filter rejects entries that lean against the prevailing direction. SCALP_TREND_FILTER defaults to true.

07 Exit Logic — Priority Ordered

P1
PSAR Stop Loss

Price crosses the SL level. SL tightens every candle via PSAR, but never widens. This is the primary risk management exit.

P2
Trailing Profit Lock

When peak P&L reaches ₹350, trailing activates. Exit when current P&L drops below 65% of peak (base tier). Higher tiers lock in more: ₹500→55%, ₹1000→60%, ₹3000→70%, ₹5000→80%, ₹10000→90%.

P3
SAR Flip (Reversal)

PSAR crosses to the opposite side of price. For CE, SAR moves from below to above price. For PE, SAR moves from above to below. Indicates trend reversal.

P4
Time-Stop (theta-bleed guard) v4.3.0

Auto-exit any trade that has stayed flat (|PnL| < TIME_STOP_FLAT_PTS, default 20 pts) for TIME_STOP_CANDLES consecutive candles (default 4). Prevents pure theta drain on directionless trades. Reason logged as TIME_STOP.

P5
EOD Square-off

All open positions squared off at 3:20 PM IST. No overnight holding for scalp trades.

10 VIX Market Regime Filter

VIX ≤ 16ALL signals allowed
16 – 20STRONG only
VIX > 20ALL blocked

VIX ≤ 16 — Calm Market

Low volatility, trending conditions. All BB breakout signals are allowed. Ideal for scalping — tight spreads, predictable moves.

VIX 16–20 — Elevated

Moderate volatility. Only "STRONG" signals pass (higher-conviction setups). Filters out marginal breakouts that might fail in choppy conditions.

VIX > 20 — High Vol

Panic/fear zone. All scalp entries blocked. Wide swings cause whipsaws, inflated option premiums erode edge. Wait for VIX to cool.

Per-Module Override v4.3.0

Scalp now carries its own VIX thresholds independent of swing. If SCALP_VIX_MAX_ENTRY or SCALP_VIX_STRONG_ONLY is unset, scalp falls back to the swing values (VIX_MAX_ENTRY / VIX_STRONG_ONLY). Toggle the entire scalp filter on/off via SCALP_VIX_ENABLED.

Note: The VIX filter is independent for scalp mode (SCALP_VIX_ENABLED). It uses the shared VIX_MAX_ENTRY (default 20) and VIX_STRONG_ONLY (default 16) thresholds from the global VIX service.

14 Key Design Principles

No Fixed Targets

There are no hard profit targets. All exits are dynamic — PSAR trail, profit lock tiers, or SAR flip. This lets winning trades run as far as the trend carries, rather than cutting winners short at an arbitrary level.

Tightening-Only SL

The stop loss can only move in one direction: toward the current price (tightening). For CE trades, SL only moves up. For PE trades, SL only moves down. This ensures that as a trade progresses favorably, the protected floor keeps rising — never giving back ground.

Multi-Layer Safety

Eight independent safety layers protect capital at trade, session, and regime levels. No single failure point can cause catastrophic loss. Each layer operates independently, and any one of them can trigger an exit or prevent an entry.

Indicator Caching

BB, RSI, and PSAR values are cached and only recalculated when candle data changes. This avoids redundant computation on every WebSocket tick during intra-candle updates, keeping CPU usage minimal during high-frequency polling.
