1 Strategy Overview

PRICE_ACTION_5M reads raw price structure on 5-minute NIFTY candles. No lagging indicators as the primary signal — patterns lead, RSI confirms.

Why 5-Minute Candles?

  • 3x more opportunities vs 15-min timeframe
  • Cleaner patterns — engulfing and pin bars resolve faster
  • Reasonable SL — typically 10-20 pts per trade
  • Zero indicator lag — reads what price IS doing
  • S/R zones from dynamic swing highs/lows
  • ADX filter blocks trades in choppy markets

2 Support & Resistance Detection

S/R levels derived from swing highs and swing lows within the last 30 candles. Each level creates a zone of +/- 15 points.

Swing Point Rules

  • Swing High: Candle whose high is greater than both its neighbors' highs
  • Swing Low: Candle whose low is less than both its neighbors' lows
  • Lookback: PA_SR_LOOKBACK = 30 candles
  • Zone Width: PA_SR_ZONE_PTS = 15 points above and below

How S/R Is Used

  • Engulfing patterns require S/R confluence
  • Pin bars (Hammer/Shooting Star) require S/R confluence
  • Inside Bar, BOS, Double Top/Bottom, Triangles do NOT require S/R
  • S/R zones re-calculate every new candle

4 ADX Trend Filter NEW

The Average Directional Index (ADX) with period 14 acts as a market regime filter, blocking all entries in choppy, trendless conditions.

How It Works

  • ADX(14) calculated on every candle from high/low/close arrays
  • Minimum threshold: PA_ADX_MIN = 20
  • When ADX < 20 — market is ranging/choppy — ALL entries blocked
  • When ADX ≥ 20 — market is trending — patterns allowed to fire
  • Can be disabled: PA_ADX_ENABLED = false

Inside Bar + ADX Interaction

  • If an inside bar is pending and ADX drops below 20, the pending state is preserved (not immediately cancelled)
  • A wait counter increments each candle while ADX is low
  • After 3 candles in low-ADX, the pending inside bar resets
  • This prevents stale breakouts in extended chop periods

Each pattern has a specific SL source. Raw SL distance is clamped to a range of 8-25 points to balance risk and noise.

Descending Triangle — Falling swing high STRONG

SL Distance Clamping

Formula: SL distance = max(min(raw_distance, 25), 8)

Min SL Floor: 8 pts

PA_MIN_SL_PTS = 8 — Prevents noise stops. If raw SL is only 3 pts away, it widens to 8 pts to avoid getting stopped on normal volatility.

Max SL Cap: 25 pts

PA_MAX_SL_PTS = 25 — Limits risk per trade. If the signal candle has a 40pt wick, SL caps at 25pts from entry.

6b Candle Trail — N-Bar Structure Exit NEW

An additional trailing mechanism that tracks the lowest low (CE) or highest high (PE) of the last N candles. Runs in parallel with the swing-based trailing SL and profit-lock floor.

How It Works

  • Enabled by default: PA_CANDLE_TRAIL_ENABLED = true
  • Lookback: PA_CANDLE_TRAIL_BARS = 2 (configurable 1-5)
  • Activates only after position has been held for N+1 candles AND peak PnL reaches PA_TRAIL_START
  • Recalculates on every candle close — level only tightens, never widens

Trail Logic per Side

  • CE trade: Trail level = lowest low of last N bars. Exit if NIFTY spot drops below this level
  • PE trade: Trail level = highest high of last N bars. Exit if NIFTY spot rises above this level
  • Checked on every tick, just like SL hit
  • Works alongside profit-lock trailing — whichever triggers first wins

Candle Trail vs Swing Trail

Swing Trail uses swing structure (pivot lows/highs) and updates less frequently — best for capturing large moves. Candle Trail uses raw N-bar extremes and updates every candle — faster to react, tighter stops. Both run in parallel; the one that triggers first closes the position.

7 Exit Logic — Priority Order

Exits are checked in strict priority. The first condition that triggers wins.

1

SL Hit (every tick)

Price touches initial SL or trailed SL. Checked on every incoming tick for immediate protection.

2

Trailing Profit % (every tick)

Tiered trailing percentage locks in profits as they grow:

₹500 → 55% ₹1000 → 60% ₹3000 → 70% ₹5000 → 80% ₹10000 → 90%
3

Candle Trail Exit (every tick) NEW

N-bar lowest low (CE) / highest high (PE) trail. Activates after peak PnL reaches trail start. Checked every tick.

4

Swing Trail SL Update (candle close)

At each candle close, tighten SL to the latest swing point. Only tightens — never widens.

5

EOD Square-off (3:20 PM IST)

All open positions closed at market. No overnight risk for intraday options strategy.

8 RSI as Confluence Filter

RSI(14) is NOT the primary signal. Patterns lead, RSI confirms. It prevents entries against momentum extremes.

CE Entry — RSI Gate

RSI > 45
  • RSI > 45: Not deeply bearish — momentum supports upward move
  • No upper cap by default. Enable PA_RSI_CAPS_ENABLED=true to add cap at PA_RSI_CE_MAX=85
  • RSI 46 is perfectly fine for CE — the pattern IS the signal

PE Entry — RSI Gate

RSI < 55
  • RSI < 55: Not deeply bullish — momentum supports downward move
  • No lower floor by default. Enable PA_RSI_CAPS_ENABLED=true to add floor at PA_RSI_PE_MIN=15
  • RSI 54 is fine for PE — the pattern provides the conviction
Key Insight

RSI acts as a veto, not a trigger. A perfect bullish engulfing at support with RSI = 75 is blocked (overbought). A mediocre pattern with RSI = 50 passes. The pattern provides the trade thesis; RSI prevents counter-momentum trades.

  • Stale cache returned on API failure — graceful degradation

8c Position Persistence — Crash Recovery NEW

Active positions are persisted to disk so they survive PM2 restarts, OOM kills, and unexpected crashes.

How It Works

  • State saved to ~/trading-data/.active_pa_position.json
  • Atomic writes via tmp + rename — no corrupted files on crash
  • Saves: side, symbol, qty, entry price, SL, best price, order ID, option details
  • Cleared automatically when position closes

Stale Position Guard

  • Each save records the current IST date
  • On boot, if the saved date is not today, the file is discarded — stale positions from yesterday are invalid for intraday
  • Prevents phantom recovery of expired positions
  • All three modes have independent persist files (Trade, Scalp, PA)

14 Practical Tips

Recommendations for deploying and tuning the Price Action strategy.

Getting Started
  • Start with backtest: /pa-backtest
  • Paper trade for 2+ weeks: /pa-paper
  • Validate pattern hit rate before going live
  • Monitor ADX values — if your market regime rarely exceeds 20, consider lowering the threshold
S/R Zone Tuning
  • Too few signals: Increase PA_SR_ZONE_PTS to 20-25 pts
  • Too many false signals: Decrease to 10 pts
  • Wider zones = more trades but lower precision
  • Tighter zones = fewer trades but higher conviction

Important Reminders
  • The ADX filter is a market regime gate, not a directional signal. ADX = 25 says "the market is moving" — not which direction. Disabled by default (PA_ADX_ENABLED=false).
  • Inside Bar is the only pattern that does NOT need S/R AND works in ranging conditions (before ADX filter was added).
  • Double Top/Bottom and Triangles are disabled by default (PA_CHART_PATTERNS_ENABLED=false). Enable them explicitly after backtesting.
  • RSI upper/lower caps are off by default. Enable PA_RSI_CAPS_ENABLED=true to block CE at RSI > 85 and PE at RSI < 15.
  • Candle Trail and Swing Trail run in parallel — whichever exit triggers first wins. Candle Trail is faster (N-bar extremes); Swing Trail is structural.
  • VIX filter has a fail-closed default — if VIX data is unavailable, entries are blocked. Set VIX_FAIL_MODE=open to reverse this.
