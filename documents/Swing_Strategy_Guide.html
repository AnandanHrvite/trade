01 — Strategy Overview

A deliberately simple, chart-driven swing strategy. Three things decide entries — an EMA alignment, an RSI band, and the SuperTrend side — and the stop is seeded at the previous candle's extreme, then trailed to EMA21 candle-by-candle. The same getSignal() drives Backtest, Paper, Live, Replay and the Live Harness, so all modes agree by construction.

In one line

CE = EMA20 above EMA50 & RSI in the CE band & SuperTrend bullish. PE = mirror. Initial stop = previous candle low (CE) / high (PE); thereafter the SL trails to EMA21 (tighten-only). Exits: trailed-SL · EMA21 touch-back · option-stop · opposite signal · exit-before-close · EOD.

There is no Parabolic SAR (removed 2026-06-12), no strength tier, no ADX, no EMA30, no candle-body or distance gate, and no 3-tier trail. Breakeven is wired but currently inert. EMA9 is only an entry input when the optional triple-stack toggle is on; otherwise it is a chart overlay (see section 12).

04 — Entry Logic — 3 Conditions

Re-evaluated on every live tick while flat (intra-candle); on 15-min it is also checked at candle close. All three conditions must be true. The EMA gate is a cross-state check — it fires while the alignment holds, not only on the bar that crosses.

BUY_CE — long call
Bullish
1. EMA alignment bullish → EMA20 > EMA50 (triple-stack: EMA9 > EMA20 > EMA50)
2. RSI_CE_MIN < RSI < RSI_CE_MAX (above momentum floor, below overbought cap)
3. SuperTrend bullish → trend = 1 (line below price)
→ Initial SL = previous candle LOW
BUY_PE — long put
Bearish
1. EMA alignment bearish → EMA20 < EMA50 (triple-stack: EMA9 < EMA20 < EMA50)
2. RSI_PE_MIN < RSI < RSI_PE_MAX (above oversold floor, below momentum cap)
3. SuperTrend bearish → trend = -1 (line above price)
→ Initial SL = previous candle HIGH
Defaults: RSI_CE_MIN=52, RSI_CE_MAX=70, RSI_PE_MAX=48, RSI_PE_MIN=30. The overbought/oversold caps stop you chasing exhausted moves. The EMA gate defaults to the 2-EMA cross (SWING_EMA_TRIPLE_STACK_ENABLED=false); set it true to require the EMA9>20>50 stack. Entry price = current tick LTP.

05 — Stop Loss & Trailing

EMA21 trailing stop (seeded at the previous candle's extreme)

Initial SL = the prior completed candle's low (CE) / high (PE), used as-is — no cap, no floor.

Trail: at every candle close the SL is moved to the current EMA21 (OHLC4) value, tighten-only (higher for CE / lower for PE; it never loosens). The new level is enforced intra-candle (tick-by-tick) during the next bar; in Live it is also pushed to the broker hard-SL when HARD_SL_ENABLED.

Optional candle trail: with SWING_CANDLE_TRAIL_ENABLED=true, an N-bar low (CE) / high (PE) over SWING_CANDLE_TRAIL_BARS bars is also computed, and whichever of (EMA21, candle level) is tighter wins.

Breakeven: the scaffolding remains but is currently inert in the swing engine — there is no active BREAKEVEN_PTS snap (that key only drives the separate Manual Tracker). A hard spot-points cap SWING_STOP_LOSS_PTS (default 25) is checked first, so a deep adverse excursion is clipped before the structural/trail SL when the prev-candle stop sits wider; set it to 0 to disable.

Because the SL hugs EMA21, a candle whose range touches back through EMA21 is treated as an explicit exit (see §6). Expect frequent small stop-outs in chop; tighter timeframes amplify this.

06 — Exit Logic

Checked per tick (price/option stops) and per candle close (trail / touch-back / signal / EOD):

#ExitRule
1Trailed-SL hitIntra-candle: CE ltp ≤ SL · PE ltp ≥ SL (SL = EMA21 trail, seeded at prev-candle extreme)
2Option-premium stopExit if option LTP ≤ entryLtp × (1 − OPT_STOP_PCT) (default 25%)
3EMA21 touch-backAt candle close, if the candle range straddles EMA21 (low ≤ EMA21 ≤ high) → exit. Skipped on the entry bar
4Opposite signalHolding CE and a BUY_PE fires → exit (and vice-versa)
5Exit before closeSquare off any open position at SWING_EOD_EXIT_TIME (default 14:30)
6EOD auto-stopAt TRADE_STOP_TIME (15:30) — square off + stop the engine
The legacy time-stop is inert (only fires under the disabled SWING_SL_MODE=candle path; the default ema trail owns the SL). In Backtest there is no live option LTP, so the option-stop is approximated as an equivalent adverse spot move (OPT_STOP_PCT × est-premium ÷ delta, BACKTEST_DELTA≈0.55); it is disabled for futures.

07 — Same-Side Cooldown

After an SL hit or option-stop on a side, new entries on that side are blocked for SWING_SL_PAUSE_CANDLES candles (default 2). The other side is free. Set to 0 to disable. This prevents immediately re-entering the same losing setup tick-after-tick.

Separately, an opposite-side cooldown (SWING_OPPOSITE_SIDE_COOLDOWN_ENABLED, default on; SWING_OPPOSITE_SIDE_COOLDOWN_CANDLES, default 2) throttles immediate CE↔PE flips. An optional chop guard (SWING_MAX_CONSEC_LOSSES, default 0 = off) halts the session after N consecutive losses.

11 — Expiry & Live Gating

  • 0DTE refusal: live/paper /start is blocked when the configured expiry == today. Replay bypasses this (force=1) so historical sessions aren't aborted.
  • TRADE_EXPIRY_DAY_ONLY — when on, only trade on NIFTY expiry day.
  • SWING_OPTION_EXPIRY_OVERRIDE / SWING_OPTION_EXPIRY_TYPE let Swing run a different expiry from the other strategies; blank inherits the common expiry.
  • Live orders are double-gated: SWING_LIVE_ENABLED=true and global LIVE_HARNESS_DRY_RUN=false, with a per-strategy SWING_LIVE_DRY_RUN override to keep Swing simulated while others trade live.
