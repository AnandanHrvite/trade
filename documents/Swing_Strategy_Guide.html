01 — Strategy Overview

A deliberately simple, chart-driven swing strategy. Four gates decide entries — an EMA alignment, an RSI band, the SuperTrend side, and a close beyond the base EMA — then a confirmation candle times the fill, and the stop is seeded at the previous candle's extreme and trailed to EMA21 candle-by-candle. The same getSignal() drives Backtest, Paper, Live, Replay and the Live Harness, so all modes agree by construction.

In one line

CE = EMA20 above EMA50 & RSI in the CE band & SuperTrend bullish & the signal candle closes above the base EMA. PE = mirror. By default a confirmation candle arms on the signal bar's close and the fill fires intra-bar when the next candle crosses that close. Initial stop = previous candle low (CE) / high (PE); thereafter the SL trails to EMA21 (tighten-only), optionally tightened by an N-bar candle trail. Exits: points-stop · option-stop · trailed-SL · negative-candle stop · EMA21 touch-back · opposite signal · exit-before-close · EOD.

There is no Parabolic SAR (removed 2026-06-12), no strength tier, no ADX, no EMA30, no candle-body or distance gate, and no 3-tier trail. Breakeven-to-entry was removed — only a dead code comment remains. EMA9 is only an entry input when the optional triple-stack toggle is on; otherwise it is a chart overlay (see section 12).

04 — Entry Logic — 4 Conditions

All four gates must agree on a fully-closed bar — the signal candle. The EMA gate is a cross-state check (it holds while the alignment does, not only on the crossing bar). By default the confirmation candle (SWING_CONFIRM_CANDLE_ENABLED, on) means entry does not fire on the signal candle: the engine arms on its close, then the fill fires intra-bar the moment the next candle strictly crosses the signal candle's close (CE above / PE below). With the toggle off it reverts to legacy intra-candle entry on the signal bar itself.

BUY_CE — long call
Bullish
1. EMA alignment bullish → EMA20 > EMA50 (triple-stack: EMA9 > EMA20 > EMA50)
2. RSI_CE_MIN < RSI < RSI_CE_MAX (above momentum floor, below overbought cap)
3. SuperTrend bullish → trend = 1 (line below price)
4. Signal candle close above the base EMA (SWING_CLOSE_BEYOND_EMA_ENABLED; base = EMA9 when triple-stack on, else EMA20)
→ Initial SL = previous candle LOW
BUY_PE — long put
Bearish
1. EMA alignment bearish → EMA20 < EMA50 (triple-stack: EMA9 < EMA20 < EMA50)
2. RSI_PE_MIN < RSI < RSI_PE_MAX (above oversold floor, below momentum cap)
3. SuperTrend bearish → trend = -1 (line above price)
4. Signal candle close below the base EMA (SWING_CLOSE_BEYOND_EMA_ENABLED; base = EMA9 when triple-stack on, else EMA20)
→ Initial SL = previous candle HIGH
Code-default RSI band: RSI_CE_MIN=52, RSI_CE_MAX=80, RSI_PE_MAX=48, RSI_PE_MIN=20 (the Settings UI seeds a tighter RSI_CE_MAX=70 / RSI_PE_MIN=30). The overbought/oversold caps stop you chasing exhausted moves. Gate 4 (SWING_CLOSE_BEYOND_EMA_ENABLED, on) requires the signal candle to close on the trade side of the base EMA — it blocks buying dips that close back under the fast EMA while the lines stay stacked from an earlier move. The EMA gate defaults to the 2-EMA cross (SWING_EMA_TRIPLE_STACK_ENABLED=false); set it true to require the EMA9>20>50 stack. With the confirmation candle on, entry price = the tick LTP at the confirming cross.

05 — Stop Loss & Trailing

EMA21 trailing stop (seeded at the previous candle's extreme)

Initial SL = the prior completed candle's low (CE) / high (PE), used as-is — no cap, no floor.

Trail: at every candle close the SL is moved to the current EMA21 (OHLC4) value, tighten-only (higher for CE / lower for PE; it never loosens). The new level is enforced intra-candle (tick-by-tick) during the next bar; in Live it is also pushed to the broker hard-SL when HARD_SL_ENABLED.

Optional candle trail: with SWING_CANDLE_TRAIL_ENABLED=true, an N-bar low (CE) / high (PE) over SWING_CANDLE_TRAIL_BARS bars (default 3) is also computed, and whichever of (EMA21, candle level) is tighter wins.

Negative-candle stop (SWING_NEG_CANDLE_LIMIT, default 2, 0=off): if a trade is still in the red (option premium below entry) at the close of N candles, it is squared off — an asymmetric loss-cut that cuts losers early while winners keep riding the trail. A hard spot-points cap SWING_STOP_LOSS_PTS is checked first on every tick, so a deep adverse excursion is clipped before the structural/trail SL when the prev-candle stop sits wider; 0 disables it.

Breakeven-to-entry was removed — there is no active breakeven snap in the swing engine (only a dead code comment remains; BREAKEVEN_PTS drives the separate Manual Tracker only).

Because the SL hugs EMA21, a candle whose range touches back through EMA21 is treated as an explicit exit (see §6). Expect frequent small stop-outs in chop; tighter timeframes amplify this.

06 — Exit Logic

Two precedence groups: per-tick (first hit wins) and candle-close (evaluated in order at each bar close):

#ExitRule
1per-tick Points stopChecked first: exit once spot has moved SWING_STOP_LOSS_PTS against entry (catastrophic cap; 0 = off)
2per-tick Option-premium stopExit if option LTP ≤ entryLtp × (1 − OPT_STOP_PCT) (default 25%)
3per-tick Trailed-SL hitCE ltp ≤ SL · PE ltp ≥ SL (SL = EMA21 trail, seeded at prev-candle extreme; optional N-bar candle trail, tighter wins)
4candle-close Negative-candle stopIf still red (option premium below entry) after SWING_NEG_CANDLE_LIMIT candles (default 2) → square off. Winners ride the trail. 0 = off
5candle-close EMA21 touch-backIf the candle range straddles EMA21 (low ≤ EMA21 ≤ high) → exit. Skipped on the entry bar
6candle-close Opposite signalHolding CE and a BUY_PE fires → exit (and vice-versa)
7candle-close Exit before closeSquare off any open position at SWING_EOD_EXIT_TIME
8candle-close EOD auto-stopAt TRADE_STOP_TIME (15:30) — square off + stop the engine
The legacy time-stop is inert (only fires under the disabled SWING_SL_MODE=candle path; the default ema trail owns the SL). In Backtest there is no live option LTP, so the option-stop is approximated as an equivalent adverse spot move (OPT_STOP_PCT × est-premium ÷ delta, BACKTEST_DELTA≈0.55); it is disabled for futures.

07 — Same-Side Cooldown

After an SL hit or option-stop on a side, new entries on that side are blocked for SWING_SL_PAUSE_CANDLES candles (code default 3; Settings-UI seed 2). The other side is free. Set to 0 to disable. This prevents immediately re-entering the same losing setup tick-after-tick.

Separately, an opposite-side (flip) cooldown (SWING_OPPOSITE_SIDE_COOLDOWN_ENABLED, default on; SWING_OPPOSITE_SIDE_COOLDOWN_CANDLES, code default 3 / UI seed 2) blocks a flip to the OPPOSITE side for N candles after any non-flip exit (SL / trail SL / option-stop / EMA touch-back) — it kills whipsaw CE↔PE flip-clusters on chop (historically the single biggest drag-reducer for Swing). Opposite-signal / EOD / manual exits do not trigger it. An optional chop guard (SWING_MAX_CONSEC_LOSSES, default 0 = off) halts the session after N consecutive losses.

11 — Expiry & Live Gating

  • 0DTE refusal: live/paper /start is blocked when the configured expiry == today. Replay bypasses this (force=1) so historical sessions aren't aborted.
  • TRADE_EXPIRY_DAY_ONLY — when on, only trade on NIFTY expiry day.
  • SWING_OPTION_EXPIRY_OVERRIDE / SWING_OPTION_EXPIRY_TYPE let Swing run a different expiry from the other strategies; blank inherits the common expiry.
  • Live orders are double-gated: SWING_LIVE_ENABLED=true and global LIVE_HARNESS_DRY_RUN=false, with a per-strategy SWING_LIVE_DRY_RUN override to keep Swing simulated while others trade live.
