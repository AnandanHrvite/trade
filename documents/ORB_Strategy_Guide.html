⚙ 1. Strategy Overview

Opening Range Breakout (ORB) is a directional intraday strategy that defines the day's "opening range" as the high/low of the first N-minute window after the market open (default 09:15 → 09:30 — the standard 15-minute ORB). Once the range is closed, the strategy waits for a candle whose close breaks above the Opening Range High (ORH) for a CE entry or below the Opening Range Low (ORL) for a PE entry. Single-leg buy — no paired hedge.

ORB is a momentum-confirmation strategy, not a fade. The thesis: directional days often reveal themselves by the first hour. If price has built a real range and then committed beyond it with a clean candle (decent body, aligned with VWAP, adequate volume), the move tends to extend in that direction through the morning session. If the breakout candle is wicky, against VWAP, or on thin volume, the breakout is more likely a fakeout and is skipped.

Why “textbook 1 trade per day”

The textbook ORB convention is one entry per session. The reasoning is that the opening range is a once-per-day structural feature — a second breakout later in the day is usually a different setup entirely (range expansion, news, late momentum). Allowing multiple entries tends to invite chop into the P&L distribution. The default is ORB_MAX_DAILY_TRADES=1; you can raise it via Settings if you accept the chop.

🕔 2. Opening Range Window

The opening range is defined by two times and observed in real-time as the session opens. Once the window closes, ORH/ORL are frozen for the rest of the day.

VariableDefaultDescription
ORB_RANGE_START09:15Opening range starts (IST). Aligns with NSE market open.
ORB_RANGE_END09:30Opening range closes. ORH/ORL are now frozen for the session.
ORB_ENTRY_END12:00Stale-breakout cutoff. No new entries past this time even if ORH/ORL is broken.
ORB_FORCED_EXIT15:15Hard square-off for any open ORB position regardless of P&L.
ORB_EXPIRY_DAY_ONLYfalseWhen true, only allow ORB entries on weekly-expiry day (Tuesday).

Building the Range

From ORB_RANGE_START until ORB_RANGE_END, the strategy tracks the highest high (ORH) and lowest low (ORL) across all candles in the window. Skip states until the range is fully formed:

  • reason: "Building opening range (waiting for 09:30 IST)" — window still open
  • reason: "Opening range not yet formed" — warming up; not enough candles to compute ORH/ORL yet
  • reason: "Warming up (X/4 candles)" — less than 4 candles available; need indicators to settle

Width Sanity Check — ORB_MIN_RANGE_PTS / ORB_MAX_RANGE_PTS

Once the range is formed, its width (ORH − ORL) is sanity-checked before any breakout can be taken.

  • Range too tight (width < ORB_MIN_RANGE_PTS, default 25 pts) → likely noise / pre-market indecision. Skip the day.
  • Range too wide (width > ORB_MAX_RANGE_PTS, default 120 pts) → open already moved; the breakout is chasing an exhausted setup. Skip the day.
Sweet spot (sub-tiering): ranges between ORB_SWEET_MIN=30 and ORB_SWEET_MAX=80 pts are considered the "sweet spot" and qualify the breakout as STRONG when combined with breakout body ≥ ORB_STRONG_BODY=15 pts. Ranges outside the sweet spot but still inside the min/max band fire as MARGINAL — still allowed, just classified differently for P&L attribution.

▶ 3. Entry Logic — 9 Gate Pipeline

A candidate breakout candle goes through 9 sequential gates. All must pass for an entry to fire. The first failing gate is recorded in the skip log with its specific reason string — useful for diagnosing "why didn't we enter today?" via /trade-logs.

1
Time Window
Candle closed after ORB_RANGE_END AND before ORB_ENTRY_END
09:30 ≤ lastIst < 12:00
2
Range Width
Opening range width within configured band
ORB_MIN_RANGE_PTS ≤ (ORH − ORL) ≤ ORB_MAX_RANGE_PTS
3
Breakout Confirmation
Candle closes beyond the range — close, not just a high/low touch
CE: close > ORH  |  PE: close < ORL
4
Min Breakout Body
The breakout candle must commit — doji-thin bodies don't count
|close − open| ≥ ORB_MIN_BODY (default 8 pts)
5
Wick Rejection Filter
Opposing wick must not dominate the body — a long upper wick on a CE break = exhaustion signal
CE: upper_wick ≤ ORB_MAX_WICK_RATIO × body  |  PE: lower_wick ≤ ...
6
VWAP Alignment
Trade in the direction the institutional flow is pricing — CE only above session VWAP, PE only below
CE: close > VWAP  |  PE: close < VWAP  (falls back to TWAP on volumeless candles)
7
Volume Confirmation
OFF by default (ORB_VOL_FILTER_ENABLED=false). When enabled, breakout volume must materially exceed the recent average — but NIFTY spot has no real volume (paper/live see a tick count, backtest sees zero), so the gate can't agree across modes and is disabled.
vol ≥ ORB_VOL_MULT × avg(prior N=ORB_VOL_LOOKBACK candles)
8
Premium-Range Gate
ATM option LTP must sit inside the configured band — filters out deep-OTM lottery tickets and ITM-acting-like-futures
ORB_PREMIUM_MIN ≤ ATM_LTP ≤ ORB_PREMIUM_MAX (default 80 — 250 ₹)
9
VIX Regime
Optional — block entries when implied volatility is too high (premiums pumped, R:R deteriorates)
VIX ≤ ORB_VIX_MAX_ENTRY (default 22)  |  STRONG-only above ORB_VIX_STRONG_ONLY (18)
Daily-count gate (pre-filter): Before any of the gates evaluate, the strategy first checks ORB_MAX_DAILY_TRADES. With the default of 1, after the first successful entry the strategy skips all further candles with reason: "Already traded this session — ORB takes only 1 trade/day".
Where each gate lives: gates 1–7 run inside orb_breakout.js getSignal() (pure signal). The premium-range gate (8) and VIX gate (9) are applied after the signal fires, at the route layer in orbPaper.js / orbLive.js — so their skip-log entries appear once a valid breakout has already been detected. Backtest seeds a fixed entry premium (ORB_BT_SEED_PREMIUM, default 180) and ignores VIX entirely; it still evaluates the premium-range gate, but against that seeded premium — so the gate acts as a config sanity check (it skips every trade if the seed lies outside the band) rather than a per-trade filter.

⭐ 4. Sweet-Spot Tiering — STRONG vs MARGINAL

Every passing breakout is classified as STRONG or MARGINAL based on the range width and breakout body. Both classifications fire entries by default — the tiering is for P&L attribution and for the VIX STRONG_ONLY gate (under elevated VIX, only STRONG signals are allowed).

ClassConditionInterpretation
STRONG ORB_SWEET_MIN ≤ range ≤ ORB_SWEET_MAX
AND body ≥ ORB_STRONG_BODY
AND volume strong (no vol-data, or vol ≥ 1.25× the vol-gate threshold ≈ 1.5× avg)
AND wick clean (wick ratio ≤ 0.6× ORB_MAX_WICK_RATIO ≈ 0.36)		 Range built in the sweet spot (30–80 pts), a real commitment body (≥ 15 pts), confirming volume and a clean (low-wick) candle. Highest-quality setup.
MARGINAL All entry gates passed but at least one STRONG sub-condition missed:
• range outside sweet spot (but inside min/max)
• body < ORB_STRONG_BODY
• volume only marginally above average
• wick ratio above the strict 0.36 bound		 Valid breakout, but the setup is less optimal. Will be blocked when VIX > ORB_VIX_STRONG_ONLY.

Entry-Reason Format

The skip / entry log on each candle uses a consistent format that surfaces all 9 gate values + tier:

ORB CE break (close 24350 > ORH 24325, range=45pt, body=18pt, wick=0.32, vwap=24310, vol=1.8×) [STRONG]

When you analyse a session log, this fingerprint identifies the trade as ORB unambiguously. See the signal-signatures memory note for cross-strategy disambiguation.

🚫 5. Exit Model — Candle-Structure Trailing Stop

Rewritten 2026-06-19 (commit 0c7f3bf). The old two-target / premium-%-stop / spot-OR-edge / move-to-breakeven / premium-lock-in model was removed in full and replaced by the single candle-structure trailing stop below. If you remember the “+40% target, −25% premium SL, +5% lock-in floor” behaviour, that code is gone — see §6 for the complete removed list.

ORB now has exactly one stop: a candle-structure trailing stop that follows the swing of the last N closed candles. There is no fixed profit target, no premium %-based stop, no opposite-OR-edge spot stop, no move-to-breakeven, and no premium lock-in. A winner simply rides until price breaks structure (or the 15:15 EOD square-off fires).

VariableDefaultWhat it controls
ORB_SL_CANDLES 2 Number of most-recent closed candles whose swing defines the stop. CE stop = the lowest low of the last N candles; PE stop = the highest high.

How the stop is set and trailed

  • Initial SL (at entry): the swing of the last ORB_SL_CANDLES closed candles — CE → lowest low, PE → highest high. This is both initialSlSpot and slSpot.
  • Trail (on every candle close): _updateCandleTrail() re-reads the swing of the last N candles and ratchets the SL in the favourable direction only — CE raises the SL to a higher low (never lowers it), PE drops the SL to a lower high (never raises it). The stop never loosens.
  • Trigger (on every tick): the exit is checked per-tick on spot in _checkExits() — CE exits when spot ≤ slSpot, PE exits when spot ≥ slSpot. The fill is the option LTP at that tick.
Update cadence vs. trigger cadence. The trail level moves only at candle close, but it is tested on every incoming spot tick. So the SL sits still through a candle, then steps up (CE) / down (PE) when the candle closes with a more favourable structure — and any tick that pierces the current level closes the trade immediately. The same level serves as both the initial stop and the trailing stop; there is no separate initial-vs-trail logic.

Worked Example — CE breakout

Breakout candle closes at 24,350 above ORH 24,325 → CE entry, spot 24,350, ORB_SL_CANDLES=2:

ENTRY  spot 24,350. Last 2 closed candle lows = 24,300 & 24,315.
       Initial SL = lowest low = 24,300 (spot).  Risk = 50 pts of spot.

Candle closes higher. Last 2 lows now 24,315 & 24,345.
       lowest low = 24,315 > 24,300  →  SL ratchets UP to 24,315.

Candle closes higher again. Last 2 lows now 24,345 & 24,372.
       lowest low = 24,345 > 24,315  →  SL ratchets UP to 24,345 (now locking gains).

Price rolls over. A tick prints spot 24,344 (≤ 24,345).
       CE EXIT on that tick, fill at the option LTP.  No premium target ever armed —
       the winner rode the structure until it broke.
Why structure, not premium. The candle stop keeps ORB in a trending move for as long as price keeps printing higher lows (CE) / lower highs (PE), instead of capping the trade at a fixed +40%. On tight-range days the stop sits close to price, so a dead-on-arrival fakeout exits fast and small rather than bleeding a wide premium stop. Note it does not rescue a trade that reverses straight off entry before any candle prints a better low/high — entry quality, not the stop, is ORB’s real lever (see §11).

📍 6. Exit Priority & the Removed Old Model

With the exit model reduced to a single stop, the runtime evaluation is short. There are only two ways an open ORB position can close:

  1. Candle-structure trailing stop (per-tick, on spot) — CE: spot ≤ slSpot  |  PE: spot ≥ slSpot. The slSpot level is ratcheted on each candle close (§5). This is the only stop.
  2. EOD forced square-off (clock, in onTick) — any position still open at ORB_FORCED_EXIT (15:15 IST default) is squared off regardless of P&L. This is the only non-stop exit — a safety net, not a strategy signal.
ORB_MAX_DAILY_LOSS does NOT kill an open trade. The daily-loss check runs in onCandleClose only when flat — the handler early-returns (if (state.position) return) the moment a position is open, and it counts realised sessionPnl only. A live ORB trade sitting at −₹800 is never force-closed by it; the position exits only on the candle trail or the 15:15 square-off. At the default 1 trade/day, this gate is effectively inert. There is no per-trade unrealised-₹ stop today — adding one would mean a new ORB_MAX_TRADE_LOSS wired into _checkExits on every tick.
ORB_TARGET_RANGE_MULT (1.5) is not an exit. It survives only as an informational chart line (targetSpot = CE: ORH + range×1.5, PE: ORL − range×1.5) drawn on the ORB status page. Nothing exits when spot reaches it.
No bid-ask spread guard on ORB exits. The shared MAX_BID_ASK_SPREAD_PTS guard is not wired into orbPaper.js / orbLive.js — ORB exits fill on the next tick without a spread check (it is a Swing/Scalp/PA guard). Confirm the spread on illiquid expiries before going live.
Removed (old exit model) — do not expect any of these:
  • −25% premium stop (ORB_STOP_PCT)
  • +40% premium target (ORB_TARGET_PCT)
  • 1.5×-range spot target as an exit (ORB_TARGET_RANGE_MULT now chart-only)
  • Opposite-OR-edge spot stop (CE: spot≤ORL / PE: spot≥ORH)
  • Move-to-breakeven ratchet (SL → entry after 1× range in favour)
  • Premium lock-in (ORB_PREMIUM_LOCKIN_PCT / ORB_PREMIUM_LOCKIN_FLOOR_PCT)
  • Continuous-premium peak-giveback trail (ORB_TRAIL_*)

All removed 2026-06-19 (commit 0c7f3bf). Their env keys are no longer read by the code (grep-confirmed) and were dropped from the Settings UI.

🔅 7. India VIX Regime Filter

ORB has its own independent VIX gate. The cross-strategy VIX_FAIL_MODE applies (when set to closed, missing VIX data blocks entries; when open, missing VIX data allows entries).

VariableDefaultEffect
ORB_VIX_ENABLEDfalseMaster toggle. When false, ORB ignores VIX entirely.
ORB_VIX_MAX_ENTRY22Above this VIX, all ORB entries are blocked — premiums are pumped, R:R deteriorates.
ORB_VIX_STRONG_ONLY18Between this and ORB_VIX_MAX_ENTRY, only STRONG signals fire (MARGINAL blocked).
Why ORB has its own VIX thresholds: ORB premiums are most sensitive to VIX of all the strategies in this suite because the trade is held intraday with a +40% target. High VIX days have higher premium ranges but disproportionately bigger drawdowns — the textbook 1-trade-per-day approach also amplifies single-day variance.

⚙ 9. Operating Modes

Paper Mode — /orb-paper

Real-time Fyers tick feed via socketManager; simulated fills (no broker round-trip). Canonical decision/fill/exit logic — the source of truth. Sessions persist to ~/trading-data/orb_paper_trades.json; per-day JSONL audit at ~/trading-data/trades/orb_paper_trades_YYYY-MM-DD.jsonl (seeded with a settings snapshot at session start, re-snapshotted on every Settings save during the session). When TICK_RECORDER_ENABLED=true (default), every spot/option/VIX tick is recorded to ~/trading-data/ticks/YYYY-MM-DD/*.jsonl for Replay.

Live Mode — /orb-live

Real Fyers orders. Double-gated: both ORB_LIVE_ENABLED=true AND LIVE_HARNESS_DRY_RUN=false are required. With DRY-RUN on (default), the broker call is logged but no real order is placed. Switch DRY-RUN off only after verifying Live and Paper decisions match on a recorded session via /replay.

Backtest Mode — /orb-backtest

Date-range backtest replaying historical 1-minute candles. Records option LTP polls during the run so Replay can reproduce the same trades deterministically. Honors LOT_MULTIPLIER end-to-end and ORB_EXPIRY_DAY_ONLY for expiry-day regime filtering. Result stored under ~/trading-data/backtest_cache/.

Replay Mode — /replay

Re-runs a recorded paper session through the paper onTick() handler. Two sub-modes:

  • Snapshot — applies the recorded session-start settings; deterministic A/A reproduction.
  • Current Settings — applies the live process.env; deterministic A/B for tuning settings against real ticks after market hours.

Replay outputs land in ~/trading-data/_replay_trades/ (snapshot) or _replay_trades_sim/ (current settings) — kept separate from the canonical paper logs.

⚠ 11. Known Gaps & Roadmap

  • Crash-recovery gap (still open). src/utils/positionPersist.js does not handle ORB positions. The other strategies (Swing / Scalp / PA) write .active_*_position.json on entry via their legacy *Live.js routes; ORB does not, so an open ORB position is not reconciled on restart. If a process crash happens with an open ORB Live position, it will be orphaned at the broker and not automatically squared off. (orbPaper.js:rehydrateSessionFromJsonl restores closed session trades from the day JSONL for display only — it is not position recovery.) Mitigation: add saveORBPosition() / loadORBPosition() / clearORBPosition() helpers and wire them into the ORB routes + app.js boot reconciliation. Until then, prefer Paper or DRY-RUN for unattended ORB sessions.
  • No per-trade unrealised-₹ stop. The candle-structure trail (§5) and the 15:15 square-off are the only exits; ORB_MAX_DAILY_LOSS guards entries only and never closes an open trade (§6). A trade can therefore sit deep red between candle closes with no rupee-cap intervention. Adding one would mean a new ORB_MAX_TRADE_LOSS checked in _checkExits on every tick.
  • Entry quality, not exits, is the loss lever. The pre-rewrite cohort (15–19 Jun, 0/4, −₹6,465) showed the losses came from dead-on-arrival false breakouts — trades that reversed straight back through the box, peak premium ≈ entry. The candle trail exits those faster and smaller, but it cannot rescue a move that never goes in favour. The real improvement is breakout confirmation / entry filtering, not further exit tuning.
  • No STRONG_ONLY standalone toggle. The strength classification exists (sweet-spot tiering), but only the VIX gate uses it. Other strategies have a stand-alone *_STRONG_ONLY env that blocks MARGINAL regardless of VIX; ORB does not. (In live data STRONG essentially never fires — every real trade has been MARGINAL.) If you want STRONG-only behaviour without VIX, this would be a small additive change in src/strategies/orb_breakout.js.
  • No bid-ask spread guard on ORB. MAX_BID_ASK_SPREAD_PTS is not wired into ORB entries or exits (see §6 / §8). On illiquid expiries an ORB exit can fill through a wide spread.
Resolved since earlier editions: Telegram per-strategy toggles are now wired — TG_ORB_STARTED / TG_ORB_ENTRY / TG_ORB_EXIT / TG_ORB_DAYREPORT exist in src/utils/notify.js (ORB emits no signal-level alerts, so there is no TG_ORB_SIGNALS).
Tuning posture: the paper-data-collection no-tuning window was lifted on 2026-05-27 — strategy tuning and bug fixes are in scope for ORB. Caveat: don't tune thresholds on trades generated before a bug fix; collect clean post-fix sessions first (in particular, discard exit behaviour from before the 2026-06-19 exit rewrite). The remaining gaps above can be addressed directly.
